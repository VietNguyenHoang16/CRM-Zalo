# Kế hoạch cải thiện dự án CRM Zalo

## Phạm vi: 🔴 Nghiêm trọng + 🟡 Trung bình (7 hạng mục)

---

## 1. Fix hydration mismatch Sidebar
**File**: `src/components/Sidebar.tsx`
**Cách làm**: Dùng `useState` khởi tạo `mounted = false`, set `mounted = true` trong `useEffect`. Chỉ render class `active` khi `mounted === true`. Giữ nguyên `isActive()` logic.

## 2. Sửa sai nhãn biến dashboard
**File**: `src/app/api/dashboard/route.ts`
**Cách làm**: 
- Đổi `pendingCall` → tách thành `pendingZalo` (PENDING_ZALO) + `pendingCall` (PENDING_CALL)
- Đổi `messagedCount` → `contactedCount` (CONTACTED)
- Thêm `messagedCount` thực sự (MESSAGED) nếu cần
- Cập nhật `src/app/page.tsx` (Dashboard) để hiển thị đúng nhãn

## 3. Thêm Error Boundary & Loading states
**Cách làm**:
- Tạo `src/app/loading.tsx` — skeleton toàn cục
- Tạo `src/app/error.tsx` — error boundary toàn cục
- Tạo `src/app/not-found.tsx` — 404 page
- Thêm try/catch vào tất cả API routes, trả về `{ error: string }` với status phù hợp

## 4. Thêm error handling cho fetch calls
**Cách làm**:
- Tạo helper `src/lib/api.ts` với hàm `apiFetch<T>(url, options?)` bọc fetch + error handling + type safety
- Cập nhật tất cả page dùng `apiFetch` thay vì `fetch` trực tiếp
- Thêm `.catch()` và hiển thị lỗi cho user

## 5. Gộp code trùng giữa Modal và Detail Page
**Cách làm**:
- Tạo `src/lib/constants.ts` — export `STATUS_LABELS`, `ACTION_LABELS`, `TEMPLATE_GROUP_LABELS` từ `utils.ts`
- Tạo `src/hooks/useCustomerDetail.ts` — custom hook chứa toàn bộ state + action logic
- Tạo `src/components/CustomerActions.tsx` — các nút hành động theo status
- Tạo `src/components/TemplatePanel.tsx` — danh sách template + chọn + copy + đánh dấu
- Tạo `src/components/CustomerTimeline.tsx` — timeline lịch sử
- Refactor `CustomerModal.tsx` và `customers/[id]/page.tsx` dùng các component chung

## 6. Thêm Zod validation cho API routes
**Cách làm**:
- Cài `zod` package
- Tạo `src/lib/validations.ts` — các Zod schema cho Customer, Action, Template, Settings
- Wrap từng API route với validation middleware pattern
- Trả về lỗi validation rõ ràng (field + message)

## 7. Thêm Prisma seed file
**Cách làm**:
- Tạo `prisma/seed.ts` với:
  - 3 CustomerSource (Facebook, Zalo, Website)
  - 5 MessageTemplate mẫu (Chào mời, Reconnect, Chăm sóc)
  - 5 Customer với các status khác nhau
  - 1 Settings mặc định
- Thêm `"prisma": { "seed": "tsx prisma/seed.ts" }` vào `package.json` (cần `tsx` dependency)
